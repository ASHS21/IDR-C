# Notifications

> For all users: stay informed about critical events and required actions.

## Prerequisites

- Signed in to Identity Radar

## Notification Bell

Click the **bell icon** in the top header to view your notifications. A badge shows the count of unread notifications.

## Notification Types

| Type | Triggered When |
|------|----------------|
| Critical Violation | A critical-severity policy violation is detected |
| Certification Due | You have entitlements awaiting your review |
| AI Analysis Complete | An AI remediation plan has been generated |
| Sync Failed | An integration source fails to sync |
| Exception Expiring | An approved exception is about to expire |
| Threat Detected | An identity threat is identified (ITDR) |

## Managing Notifications

- **Mark as Read**: Click a notification to mark it as read
- **Mark All Read**: Click the checkmark icon to clear all unread badges
- **Dismiss**: Swipe or click the dismiss button to remove a notification

## Email Preferences

Configure email notification preferences in **Settings > Notifications**:

- Enable or disable email alerts for each notification type
- Set a digest frequency (immediate, hourly, daily)
- Configure the SMTP server if not already set up

## Verification

Notifications appear in real time when critical events occur, and the unread badge count is accurate.

## Troubleshooting

| Problem | Solution |
|---------|----------|
| No notifications received | Check that notification preferences are enabled in Settings |
| Email notifications not arriving | Verify SMTP configuration in Settings. Check spam folder. |
| Badge count incorrect | Refresh the page to sync the notification state |
