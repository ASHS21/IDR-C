# Configuration Reference

> Every environment variable in `.env.local` with type, default, and usage.

## Database

| Variable | Type | Default | Description |
|----------|------|---------|-------------|
| `POSTGRES_DB` | string | `identity_radar` | PostgreSQL database name |
| `POSTGRES_USER` | string | `postgres` | PostgreSQL username |
| `POSTGRES_PASSWORD` | string | `changeme` | PostgreSQL password. Change in production. |
| `DATABASE_URL` | string | `postgresql://postgres:changeme@localhost:5432/identity_radar` | Full connection string used by Drizzle ORM |

## Authentication

| Variable | Type | Default | Description |
|----------|------|---------|-------------|
| `NEXTAUTH_SECRET` | string | (generated) | Secret key for signing JWT tokens. Generate with `openssl rand -base64 32`. |
| `NEXTAUTH_URL` | string | `http://localhost:3000` | Canonical URL of the application. Set to your domain in production. |

## AI Providers

The AI engine tries providers in order: local model (Ollama/Docker Model Runner) then Anthropic cloud.

| Variable | Type | Default | Description |
|----------|------|---------|-------------|
| `OLLAMA_URL` | string | `http://localhost:11434` | URL of the local Ollama or Docker Model Runner instance |
| `OLLAMA_MODEL` | string | `qwen2.5:7b` | Model name to use for local inference |
| `ANTHROPIC_API_KEY` | string | (empty) | Anthropic API key. Optional; used as fallback if local model is unavailable. |

## Email (SMTP)

All SMTP settings are optional. Email notifications are disabled if not configured.

| Variable | Type | Default | Description |
|----------|------|---------|-------------|
| `SMTP_HOST` | string | (empty) | SMTP server hostname |
| `SMTP_PORT` | number | `587` | SMTP server port |
| `SMTP_SECURE` | boolean | `false` | Use TLS for SMTP connection |
| `SMTP_USER` | string | (empty) | SMTP authentication username |
| `SMTP_PASS` | string | (empty) | SMTP authentication password |
| `SMTP_FROM` | string | `Identity Radar <noreply@identityradar.io>` | From address for outgoing emails |

## Application

| Variable | Type | Default | Description |
|----------|------|---------|-------------|
| `NEXT_PUBLIC_APP_URL` | string | `http://localhost:3000` | Public URL used by the frontend. Must match `NEXTAUTH_URL`. |
| `NEXT_PUBLIC_DEMO_MODE` | boolean | (empty) | Set to `true` to show the demo mode banner |

## Production (Caddy / HTTPS)

| Variable | Type | Default | Description |
|----------|------|---------|-------------|
| `DOMAIN` | string | (commented out) | FQDN for automatic Let's Encrypt HTTPS (e.g., `radar.example.com`) |

## When to Change Each Variable

- **First install**: `POSTGRES_PASSWORD`, `NEXTAUTH_SECRET` (auto-generated by setup script)
- **Production deployment**: `NEXTAUTH_URL`, `NEXT_PUBLIC_APP_URL`, `DOMAIN`, `POSTGRES_PASSWORD`
- **Azure AD integration**: No env var needed (configured in UI)
- **Email notifications**: All `SMTP_*` variables
- **Cloud AI fallback**: `ANTHROPIC_API_KEY`
